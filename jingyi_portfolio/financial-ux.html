Market Watch Web

Real-time product design


Product Brief:

This project is to design a new web version Market watch (MWW) based on the old workstation product. Besides allowing users to simultaneously monitor market data (such as equities, indices, treasuries, and futures), new MWW wants to give users more feasibility to create and organize their own data. The whole project lasts almost half a year, due to the new introduction of technology and the change of product requirements. The majority of time for design phase is about 2 months, including design research, ideation, and iteration with visual and engineering team. I was the lead designer who took responsibility for this project.


User needs mainly are:
  • Efficient real-time data manipulation
  • Flexibility to play around with data
Challenges are:
  • How to inherit the old main features but deliver more fluent UX/workflow based on users’ needs.
  • How to solve the conflicts between the standards across applications and the specific need of this product, and how to balance them and give the optimal solution.
What I mainly did:
  • “Translated” every piece of requirements into design language
  • Created the new solution for “Row and column” interaction
  • Designed the new features of “Grouping”
  • Redesigned the whole “Options” system
Design deliverables:
  • Use cases analysis
  • Interaction workflow
  • Information Architecture
  • Wireframes

Product Brief:

This project is to invent a new application crossed platform that helps users to save the financial reports they are interested onto the “clouds” so that those materials can be accessed anywhere and anytime. Cloud is my first big cross platform project for which I designed 3 different platform-based applications and 1 desktop widget. I was fully responsible for all the UX parts of this project, and work collaboratively with Product Development team and visual designers. The project lasted 3 month or so.


User needs are:
  • A simple way to save and sync financial reports across platform.
  • Efficient Content management
  • More working flexibility - a seamless cross-platform experience
Challenges are:
  • Brand new product invention with no usage data support
  • It needs to create the whole system across 3 different platforms
  • Detailed interactions are complicated.
What I designed/learned:
  • Cross platform design: Syncing data throughout all the platforms (mobile and desktop)
  • Knowing users’ busy daily life scenarios: Quick and easy bookmarking
  • Content management: ways to manage users’ files/docs – all, read, unread, download and TAG
Design solutions (deliverables):
  • Mobile design, tablet design, and desktop design
  • Small widget used for quick bookmarking (“add tag” dialog)
  • Brainstorming & Sketches
  • low/med fidelity wireframes

StreetAccout Filter Design

News and research product design


Product Brief:

This project focuses on how to filter complex information and improve the workflow. The product involves financial news, clients’ portfolios and industrial sectors. Diving into the information architecture of the product was crucial. The whole project lasts a quarter. I worked with PD and PD’s engineer very closely, and got great insights from real data/facts. I was the main designer who’s responsible for this project.


User needs are:
  • An efficient filter based on portfolios and sectors filtering
  • Ability to save favorite filter set
Challenges are:
  • It takes time to fully understand the current AI (Exclusive vs. non-exclusive)
  • How to use the (raw) real data to redesign the product
  • How to clearly present the relationship between different filters and make better workflow
What I mainly did/learned:
  • Created the Information Architecture of filtering based on the requirements
  • Designed advanced “Content Filter”
  • learned that Real user data matters.
Design deliverables:
  • Information architecture diagram
  • Wireframes
